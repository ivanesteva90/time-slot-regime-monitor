Metodologia embebida

1. Ingestion, DATA y normalizacion base

El dashboard puede leer CSV subidos manualmente o la DATA guardada via manifest.json. Cada archivo se normaliza a Date, Hora y P/L, eliminando filas invalidas y compactando duplicados por promedio antes de entrar al motor analitico.

2. Forecast score, agotamiento y etiquetas

Cada slot se puntua con ventanas rolling de 10, 21, 63, 126 y 252 observaciones. El forecast prioriza robustez estructural y continuidad del slot; el agotamiento detecta deterioro reciente por perdida de rango, caida del bloque corto, slip de win rate y ratio 10d/63d. Las etiquetas KEEP / WATCH / ROTATE salen del equilibrio entre fuerza relativa y cansancio actual.

3. Walk-forward y validation health

La validacion del dataset usa un walk-forward secuencial: para cada fecha se rankean slots con solo la historia previa y luego se compara el slot elegido contra el resultado real de ese dia. De ahi salen hit rate, beat average, top 3 coverage y el validation health agregado del dataset.

4. Screener y scope compartido

El screener filtra por fechas, horas, indice, perfil y tipo. Desde esta version, Weekly Outlook, Weekday Quality y Estrategia combinada sugerida ya no miran siempre el universo completo: se recalculan con el mismo scope activo del screener para que la lectura sea coherente entre secciones.

5. Weekly Outlook V1

El outlook semanal V1 es offline. No usa live feed todavia. La probabilidad diaria mezcla historia del slot, comportamiento del weekday, tramo reciente y estado estructural actual. El EV usa medias suavizadas de ganancia y perdida; el horizonte semanal se proyecta con simulacion Monte Carlo sobre pools historicos de outcomes. La validacion muestra next-day y next-week. El modulo usa dias habiles simples; todavia no descuenta holidays ni POP teorico de mercado.

6. Weekday Quality y mix sugerido

Weekday Quality diagnostica cada dia por expectativa, profit factor, drawdown, estabilidad mensual, muestra y tendencia reciente. No asigna sizing. La estrategia combinada arma un esquema core / satellite / diversifier con los datasets mejor posicionados dentro del scope vigente, intentando repartir simbolo, estructura y perfil sin perder salud.

7. Reglas practicas de uso
  • Recalcula el tablero cada vez que actualices CSV o cambies el screener si quieres coherencia entre ranking y secciones derivadas.
  • Usa la validacion walk-forward antes de perseguir un slot o un mix que se vea demasiado bien en bruto.
  • Si el scope del screener deja poca muestra, el dashboard prioriza mostrar insuficiencia antes que inventar conclusiones.
  • El V1 del outlook sirve para escenarios y priorizacion, no para prometer P/L exacto ni para reemplazar disciplina de riesgo.
